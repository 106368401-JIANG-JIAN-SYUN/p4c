26 /*
27  * Hindley-Milner type unification algorithm
28  * (See for example, http://cs.brown.edu/~sk/Publications/Books/ProgLangs/2007-04-26/plai-2007-04-26.pdf, page 280.)
29  * Attempts to unify two types. As a consequence it generates constraints on other sub-types.
30  * Constraints are solved at the end.
31  * Solving a constraint generates type-variable substitutions
32  * (where a type variable is replaced with another type - which could still contain type variables inside).
33  * All substitutions are composed together.
34  * Constraint solving can fail, which means that the program does not type-check.
35  */
36 class TypeUnification final {
37  TypeConstraints* constraints;
